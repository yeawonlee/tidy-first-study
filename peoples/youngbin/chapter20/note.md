# Chapter 20: 얽힘 풀기
동작 변경을 하려다가, 이를 더 쉽게 할 코드 정리 방법을 먼저 알아서 이를 먼저 수행한다. 그리고 계속 동작 변경을 하려 하니, 코드정리를 더 해야할 부분이 생겼고 여기서 파생되는 동작 변경 작업을 알게 되었고... 이를 쉽게 할 코드 정리 작업도 알게 되었다. 마치 책에서 나온 것 처럼 얽힌 실타래 하나 발견한 것 같다.
이럴때 그냥 처음부터 다시 하라고 설명을 한다. 얽힌 실타래를 발견 하였고, 어떻게 얽혀 있는지도 파악을 하였으니 이를 바탕으로 처음부터 다시 코드 정리를 하고 동작 변경을 하라는 설명으로 이해가 된다. 그 이유는 컴퓨터에 지시하는 의도를 코드를 보는 다른 사람에게도 설명을 해야 하기 때문이다. 잘 돌아가고 빨리 굴러가는 코드 작성 보다, 다른 사람이 봤을 때 그 의도를 잘 알 수 있는 코드를 작성하는 것이 더 중요하기 때문이다.
