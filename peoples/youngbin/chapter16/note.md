# Chapter 16: 코드 정리 구분
코드 정리 한 것을 결국에는 코드베이스에 반영을 해야 하고, 동료와 함께 작업하는 코드라면 PR 등으로 검토를 받아서 반영을 해야 한다. 그런데 이 과정에서 가급적 검토하는 사람에게 부담이 가지 않도록 하는 내용에 대한 것으로 이해가 된다. 하나의 큰 PR 덩어리에 변경사항과 코드정리를 넣어서 제출하면 검토자는 아마도 피곤할 터, PR을 작게 하자는 누군가는 잡초같이 볼 수도 있다. 결국에는 PR 분량을 검토하는 사람에게 부담되지 않는 선에서 조정하고 그 범위 안에서 코드 정리 내용도 포함하거나, 아니면 코드 정리만 별도 PR 로 제출하는 것 등이 핵심으로 이해가 된다.
