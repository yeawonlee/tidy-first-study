# Chapter 17: 연쇄적인 정리
앞에서 다룬 내용이 전체적으로 한번씩 더 나오고, 이런 것들을 시작하다 보면 계속 하고 싶어서 연쇄적으로 코드를 정리하고 싶을 수 있다, 하지만 그러다가 한번에 많이 정리하면 실패 가능성이 높으니 작은 것 부터 하나씩 정리를 해 나가라는 내용. 개인적으로 몇년전에 잠깐 한가할 때 크게 한번 코드 정리를 한 적이 있다. 특정 디자인 패턴 적용 해 본다거나, 그런 것이였던 것 같은데, 지금 생각 해 보니 그렇게 작업 하고 테스트 해 봐서 복구 해야 하면 이것 또한 너무 번거로운 일. (물론 revert 커밋 만들거나 해서 되돌릴 수도 있겠다). 앞으로는 조금씩 해야 하겠다는 생각이 들었다.
