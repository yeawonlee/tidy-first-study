# Chapter 19: 리듬
개인이 코드 정리 한번 하는데 시간을 얼마나 들이는 것이 적절한가에 대해 생각 해 볼 수 있는 파트였다. 소프트웨어 설계 (크고 작업 코드 정리를 포함하는)는 마치 길을 닦는 작업과도 같아서, 한번에 할 때 생각 보다 오래 걸릴 수도 있고, 그렇게 하고 나서야 나머지 작업을 더 효율적으로 할 수도 있지만, 그렇다고 몇시간을 하기는 어렵고, 이 책에서는 작은 것 부터 조금씩 코드 정리를 하자는 내용이니, 그에 맞는 기준을 제시 해 준 것 같다. 책에서 나온 건물 사이 연결 통로를 굳이 처음에 안 만들고, 잔디 심어서 나중에 잔디 밟고 다녀서 자연스럽게 생긴 길을 포장 한 예시가 흥미로웠다.
소프트웨어 개발로 치면, 처음부터 미래의 여러 상황을 가정해서 신중하게 결정 해다가 코드 정리하고 구현하는것도 좋지만, 결정하기 어렵고 비용이 큰 부분은 나중에 사용자들이 사용하는 것을 보고 결정해도 된다는 것과 비슷한 것 같기도 하다.
