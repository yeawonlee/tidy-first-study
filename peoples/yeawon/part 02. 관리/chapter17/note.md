## 17장. 연쇄적인 정리

- 코드를 계속 정리하고 싶은 충동을 관리하는 것도 코드 정리의 핵심 기술이다
- 저자가 권하는 단계는, **아주 작은 단계로 나누어 코드를 정리하는 방식으로 실험해 보는 것**. 그러면서 각 단계를 최적화해라

- **보호 구문**: 보호 구문으로 정리하면, 조건이 설명하는 도우미로 드러나거나 설명하는 변수 추출을 도울 수 있다
- **안 쓰는 코드 제거**: 사용하지 않는 코드 더미를 제거하면, 코드를 읽는 순서에 맞춰 정렬하는 방법과 응집도를 높이는 배치가 보일 것이다
- **대칭으로 맞추기**: 같은 코드와 다른 코드를 일치시켜 대칭으로 맞추다 보면, 매우 유사한 코드들이 묶여진 순서대로 읽을 수 있다
- **새로운 인터페이스로 기존 루틴 부르기**: 새 인터페이스를 만들고 난 후, 자동으로 호출 코드를 바꿔주는 개발 도구가 없다면 일일이 바꿔줘야 한다. 호출하는 모든 코드를 변환할 수 있는 자동 재작성 도구가 없어도 하나씩 변환해야 한다. ... 결합도와 멱법칙에 대해 이야기할 때 더 살펴보자
- **읽는 순서**: 읽는 순서를 정리하고 나면, 대칭으로 맞출 기회가 생길 수 있다.너무 멀리 떨어져 있어 유사한 줄 몰랐던 요소들이 읽는 순서 정리로 드러나는 것
- **응집도를 높이는 배치**: 응집도를 높이는 배치로 함께 묶인 요소는 하위 요소로 추출할 후보가 된다
- **설명하는 변수**: 할당하는 문장에서 좌변이 설명하는 변수라면, 그에 대응하는 우변은 설명하는 도우미 후보일 수 있다 (나중에 도우미 추출로 인해 변수가 흡수되어 사라질 수도 있음). 변수 이름으로 불필요한 주석이 드러나 삭제할 수도 있다
- **설명하는 상수**: 설명하는 상수 정리는 응집도를 높이는 배치를 이끈다. 한 번에 바뀌는 상수는 모아서 묶어 놓으면 나중에 바꾸기 좋다
- **명시적인 매개변수**: 매개변수를 명확하게 드러나게 만들면, 매개뱐수 집합을 묶어 객체로 만들고 코드를 옮길 수 있다. 이것은 코드 정리를 넘어서는 것이지만, 코드를 정리하면서 새로운 추상화가 도출될 수 있는지 계속 주목하는 것이 좋다
- **비슷한 코드끼리**: 코드 덩어리 앞에 주석을 붙일 수 있다. 코드 덩어리를 설명하는 도우미로 바꿀 수도 있다
- **도우미 추출**: 도우미 추출 후에는 보호 구문을 도입하거나 설명하는 상수, 변수를 추출할 수도 있고, 불필요한 주석을 지울 수도 있다
- **하나의 더미**: 코드가 모여서 크고 명백하게 엉망인 모습이 보이면, 비슷한 코드끼리 정리, 설명하는 주석 정리, 도우미 추출 따위의 방법을 기대할 수 있다
- **설명하는 주석**: 설명하는 변수나 상수, 도우미 따위의 도입이 가능하다면, 주석에 있는 정보를 코드로 옮긴다
- **불필요한 주석 지우기**: 
    - 불필요한 주석에서 오는 잡음만 없애도, 읽는 순서를 개선하는 데 도움을 주고, 명시적인 매개변수를 쓰는 기회가 찾아온다. 
    - 완전히 중복된 주석만 삭제해야 한다. 또한, 주석은 오직 모두 필요한 내용으로만 정리해야 한다

### 결론
- **너무 많이, 너무 빠르게 변경하지 않도록 주의하자**
- **작은 정리**를 순차적으로 성공하는 것이 무리한 정리로 실패하는 것보다 **시간을 아낄 수 있다**
